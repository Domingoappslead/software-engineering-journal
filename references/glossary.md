# Glossary of Software Engineering Terms

A central reference for key technical terms I encounter while learning C#, .NET, API development, Blazor, and general software engineering.  
Each term includes a concise definition and (optionally) a link to a related note.

---

## A

### API (Application Programming Interface)
A structured contract that defines how software components communicate.  
Related: [notes/api/rest-fundamentals.md] (currently doesn't exist)

### Abstraction
The process of hiding internal complexity and exposing only essential details.  
Related: [notes/software-engineering/solid-principles.md] (currently doesn't exist)

---

## C

### Class
A blueprint for creating objects in C#. Defines properties, methods, and behaviors.  
Related: [notes/csharp/classes-and-objects.md] (currently doesn't exist)

### Clean Architecture
A software architecture style that separates business logic from frameworks and external dependencies.  
Related: [notes/software-engineering/clean-architecture.md] (currently doesn't exist)

---

## D

### Dependency Injection (DI)
A design pattern where required dependencies are provided by an external container instead of being created inside a class.  
Related: [notes/dotnet/dependency-injection.md] (currently doesn't exist)

### DTO (Data Transfer Object)
A lightweight object used to transfer data between layers or across the network.  
Related: [notes/api/rest-fundamentals.md] (currently doesn't exist)

---

## E

### Entity
An object with a unique identity in domain modeling (e.g., a database row with an ID).  
Related: [notes/databases/entity-framework-core.md] (currently doesn't exist)

## H

### HTTP (Hypertext Transfer Protocol)
The foundation of communication between clients and servers in web applications.  
Related: [notes/api/http-basics.md] (currently doesn't exist)

---

## M

### Middleware
A component in the .NET request pipeline that processes incoming HTTP requests and outgoing responses.  
Related: [notes/dotnet/middleware-pipeline.md] (currently doesn't exist)

---

## O

### OOP (Object-Oriented Programming)
A programming paradigm based on organizing code around objects, classes, and relationships.  
Related: [notes/csharp/oop-basics.md] (currently doesn't exist)

---

## S

### SOLID Principles
A set of five design principles for writing maintainable, scalable, and flexible object-oriented code.  
Related: [notes/software-engineering/solid-principles.md] (currently doesn't exist)

### Serialization
The process of converting an object into a format that can be stored or transmitted (e.g., JSON).  
Related: [notes/api/rest-fundamentals.md] (currently doesn't exist)

## V

### Value Type
A type that holds data directly in memory (stored on the stack).  
Related: [notes/csharp/variables-and-types.md] (currently doesn't exist)

### Validation
Rules and checks that ensure incoming data is correct and safe to process.
